### 목표 : Guacamole 세팅
  
사용하던 싸지방 컴퓨터가 고장나서 한동안 개발에서 손을 놓다가 우연히 HTML5 기반 RDP를 발견하여 세팅해보았다  
__Guacamole__ 이란것인데 아파치 기반으로 RDP,SSH,VNC등을 웹에서 지원해주는넘이다  
내가 기존에 갖고있던 GCP서버에 Docker를 이용하여 설치하였는데 웹기반인것 치고는 반응속도도 나쁘지 않아서  
파일 전송이나 클립보드 공유등의 기능도 들어있어 유용하게 사용할 수 있을 것 같다

대략적인 설치방법은 다음과 같다  
https://github.com/oznu/docker-guacamole 를 참고하여 진행하였는데  
Docker 가 설치되어있다는 전제하에  
docker run \
  -p 8080:8080 \
  -v </path/to/config>:/config \
  oznu/guacamole  

를 입력해주면 Docker 설정은 끝이다  
여기서 주의해야 할 점은 포트포워딩 인자에 8080Port가 안먹힐시 80Port를 사용해보는것과  
볼륨인자에 사용할 볼륨 경로를 정확히 입력해줘야하는 것이다  
특히 볼륨경로를 정확히 입력해주지 않으면 다른 에러로그없이 겉으로는 잘 작동중인 것 처럼 보이지만  
실제로는 접속이 불가하니 특히 주의를 해야한다  
  
이후 서버 IP로 접속시 정상적으로 잘 잘동한다면 다음과같은 화면이 뜨는데  
![Guacamole](/img/2020/11/21/guacamole.PNG)   
  
초기 IP및 Password는 __guacadmin__ 이므로 이후 알아서 바꿔주도록 하자  
  
RDP연결 기준으로 연결할 컴퓨터의 IP만 입력해줘도 기본적인 접속은 가능하나  
이전에 연결할 컴퓨터에서 원격 접속을 허용해놓고 사용시 비밀번호를 미리 걸어놓아야만 접속이 된다  
   
추가로 OWL이나 DDNS등을 설정해놓으면 더욱 사용하기 편하다  
  
다시 환경을 세팅하였으니 원래하던 개발을 이어나가야겠다
